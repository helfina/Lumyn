# Guide de développement de Lumyn

## Règles

1. Une fonctionnalité doit résoudre un problème réel.
2. Une seule étape principale est réalisée à la fois.
3. Une étape doit être comprise avant de passer à la suivante.
4. Le projet doit toujours rester dans un état fonctionnel.
5. Les décisions importantes doivent être écrites dans DECISIONS.md.
6. Chaque séance doit se terminer par la mise à jour de JOURNAL.md et PROJECT_STATE.md.
7. Le code doit être clair, documenté et testé.
8. Aucun mot de passe ni aucune clé API ne doit être enregistré sur GitHub.

## Messages de commit

Le message doit expliquer ce qui a changé.

Bon exemple :

Ajout de la documentation initiale du projet

Mauvais exemple :

Modifications
